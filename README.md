# jewellery-inventory-manager

A web application to manage inventory for a jewellery business

## Overview

This project was generated by AIxNative and contains the following modules:

- **Database Schema** (database): Prisma schema with Item, Supplier, Purchase, Sale, Customer, User models
- **Authentication** (auth): NextAuth authentication for users
- **Item Service** (service): Business logic for items
- **Supplier Service** (service): Business logic for suppliers
- **Purchase Service** (service): Business logic for purchases
- **Sale Service** (service): Business logic for sales
- **Customer Service** (service): Business logic for customers
- **Metal Rate Service** (service): Business logic for metal rates
- **Report Service** (service): Business logic for reports
- **Dashboard API** (api): REST endpoints for dashboard data
- **Item API** (api): REST endpoints for items
- **Supplier API** (api): REST endpoints for suppliers
- **Purchase API** (api): REST endpoints for purchases
- **Sale API** (api): REST endpoints for sales
- **Customer API** (api): REST endpoints for customers

## Getting Started

### Prerequisites

- Node.js 18+
- npm or yarn

```bash

```

### Running the Application


## Modules


### Database Schema

**Type:** database

Prisma schema with Item, Supplier, Purchase, Sale, Customer, User models

**Acceptance Criteria:**
- Item model with id, name, category, metalType, purity, grossWeight, netWeight, stoneDetails, makingCharges, wastage, finalPrice
- Supplier model with id, name, contactDetails
- Purchase model with id, itemId, supplierId, purchaseDate, quantity, price
- Sale model with id, itemId, customerId, saleDate, quantity, price
- Customer model with id, name, contactDetails
- User model with id, name, email, role, password
- Foreign key relationships between models
